Adapter-design verdict (per plan § Stage 1.C step 3)
================================================================================

For each RESTest import, classify as WRAP (define mist-core interface, adapter
implements against RESTest), INLINE (≤ 50-LOC standalone helper, copy into
mist-core), or KEEP (stays on the adapter side).

CONFIGURATION POJOS  (Operation, TestParameter, TestConfigurationObject, Auth,
                      TestConfiguration)
  -> WRAP. Define `io.mist.core.spec.{MistOperation, MistParameter,
     MistTestConfiguration}` records; adapter materialises them from the
     RESTest POJOs. The shapes are simple data carriers — mapping is
     straightforward.

SPECIFICATION  (OpenAPISpecification, OpenAPISpecificationVisitor,
                OpenAPIParameter, OpenAPIOperation)
  -> WRAP via `io.mist.core.spi.MistSpecLoader` interface:
     `MistOpenApiSpec loadSpec(Path path)`. The internal `MistOpenApiSpec`
     contains only the data mist-core needs (operations, schemas,
     parameters).

UTIL CLASSES
  - ConsoleProgressBar       -> INLINE (small helper).
  - SeededRandom             -> INLINE (the seed gate is core to MIST).
  - LLMCommunicationLogger   -> INLINE.
  - SchemaManager            -> WRAP via `MistJsonSchemaService`.
  - RESTestException         -> WRAP / INLINE: define `io.mist.core.MistException`.
  - PropertyManager          -> WRAP via `MistProperties` (interface).

TEST CASES
  - TestCase                 -> WRAP via `io.mist.core.api.MistTestCase` (an
     interface MultiServiceTestCase already conforms to in spirit).

INPUTS
  - InvalidInputPool         -> KEEP on the MIST side. The class IS MIST code
     in spirit. Phase 3.D moves it into io.mist.core.fault.
  - InvalidInputType         -> KEEP on the MIST side (retired by Phase 3.A).
  - inputs.stateful.ParameterGenerator -> WRAP via
     `io.mist.core.spi.MistParameterGenerator`.

COVERAGE
  - StatusCodeTarget         -> INLINE.
  - StatusCodeCoverageTracker -> INLINE.
  - LLMStatusCodeDiscovery   -> KEEP (used only by enhancer; goes into
     io.mist.core.enhancer).

WRITERS
  - RESTAssuredWriter        -> WRAP via `io.mist.core.spi.MistWriter`.
     `MultiServiceRESTAssuredWriter` becomes an adapter-side implementation.

package io.mist.core.spi;

public interface MistSpecLoader {
    MistOpenApiSpec loadSpec(Path path) throws IOException;
}

public interface MistTestConfigLoader {
    MistTestConfiguration loadConfiguration(Path path) throws IOException;
    Map<String, MistTestConfiguration> loadMultiServiceConfiguration(InputStream in) throws IOException;
}

public interface MistJsonSchemaService {
    Object resolveExampleValue(MistParameter parameter);
    boolean isValid(Object value, String oasType, String format);
}

public interface MistAllureBridge {
    void emitStep(String name);
    void attach(String name, byte[] content, String mimeType);
}

public interface MistWriter {
    void write(java.util.Collection<? extends io.mist.core.api.MistTestCase> testCases);
    void setClassName(String name);
    void setTestId(String id);
}

public interface MistProperties {
    String read(String key);
}
